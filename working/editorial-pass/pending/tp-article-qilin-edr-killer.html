Executive Summary

The Qilin ransomware group — also tracked as Agenda, Gold Feather, and Water Galura — has deployed a sophisticated, multi-stage attack chain capable of disabling endpoint detection and response (EDR) tools from virtually every major security vendor before executing its ransomware payload. The technique, documented by Cisco Talos in April 2026, represents a significant escalation in the group's operational tradecraft.

At the core of the campaign is a malicious msimg32.dll that is sideloaded via legitimate applications such as FoxitPDFReader.exe. This DLL serves as the entry point for a four-stage, memory-resident loader chain that ultimately delivers a kernel-level EDR killer capable of unregistering monitoring callbacks across more than 300 EDR driver names — effectively blinding the entire defensive layer before ransomware is ever dropped.

Qilin has reported over 40 victims per month as one of the most active ransomware-as-a-service (RaaS) operations currently in operation. This campaign demonstrates a deliberate shift toward neutralizing the defense layer as a primary phase of the attack — not an afterthought.

Attack Chain

The attack unfolds in four distinct stages, each designed to advance deeper into kernel space while avoiding detection:

Stage 1 — DLL Sideloading & Entry
→
A legitimate application (e.g., FoxitPDFReader.exe) loads a malicious msimg32.dll from its local directory instead of the genuine Windows system library. The rogue DLL forwards all expected API calls to the real C:\Windows\System32\msimg32.dll, preserving normal application behavior while executing malicious logic from DllMain.
Stages 2–3 — Anti-Detection Loader Chain
→
SEH/VEH Abuse: Structured and Vectored Exception Handling mechanisms are weaponized to conceal API invocation patterns and covertly transfer execution between loader stages.
→
ETW Suppression: Event Tracing for Windows is neutralized at runtime, depriving defenders of the behavioral telemetry needed for detection.
→
Halo's Gate Syscall Bypass: The loader scans ntdll.dll both forward and backward to locate clean, unhooked neighboring syscall stubs, then repurposes those stubs to invoke system calls — bypassing EDR-hooked APIs without modifying hooked code.
→
Kernel Object Manipulation: The .mrdata section of ntdll.dll — which holds the exception dispatcher callback pointer — is overwritten to redirect exception handling to the loader's own routine.
→
Anti-Debugging: Checks for breakpoints on KiUserExceptionDispatcher; deliberately crashes the process if one is detected. Geo-fencing also terminates execution on post-Soviet system locales.
Stage 4 — Kernel-Level EDR Killer
→
rwdrv.sys: A renamed version of ThrottleStop.sys, legitimately signed by TechPowerUp LLC. Exposes IOCTLs for physical memory read/write and MSR access — exploited here to directly manipulate kernel structures bypassing protected virtual memory.
→
hlpdrv.sys: Used exclusively to terminate protected EDR processes via IOCTL code 0x2222008, bypassing Windows process protection mechanisms.
→
Iterates through a hardcoded list of 300+ EDR driver names, using physical memory writes via rwdrv.sys to unregister process creation, thread creation, and image load monitoring callbacks — disabling EDR visibility at the kernel level across virtually all major vendors.
→
Temporarily overwrites the CiValidateImageHeader callback to bypass Code Integrity enforcement, then restores it afterward to reduce forensic traces.

Remediation & Mitigation

Cisco Talos notes that while the individual techniques are not entirely novel, they remain highly effective against single-layer defense stacks. Organizations are advised to:

Detection Indicators
→
Monitor for rwdrv.sys and hlpdrv.sys driver installation events — neither should appear in a legitimate production environment.
→
Alert on any user-mode process attempting physical memory writes — this behavior is a strong indicator of kernel manipulation.
→
Monitor FoxitPDFReader.exe (and similar LOLBin candidates) loading msimg32.dll from non-system directories.
→
Monitor for ETW suppression and unexpected changes to ntdll.dll sections in memory.
→
Alert on modifications to CiValidateImageHeader or Code Integrity callbacks.
